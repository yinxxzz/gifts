# 销售赠品策略工作台 · 同事部署说明

这个仓库是一个可独立部署的静态工作台，线上地址：

`https://gift-strategy.run.zhenguanyu.com/`

## 一、日常怎么改

常改的内容主要有三类：

1. 文案和期次：改 `data/periods.json`
2. 图片素材：放到 `assets/` 对应期次目录
3. 上架状态/在线编辑记录：改 `data/gift-shelf-status.json`、`data/period-edits.json`

改完后提交到 GitHub。

```bash
git add .
git commit -m "更新赠品策略"
git push
```

## 二、首次部署到 Run 机器

登录 Run 机器后执行：

```bash
mkdir -p /home/shared/workspace
cd /home/shared/workspace
git clone https://github.com/yinxxzz/gifts.git gift-strategy
cd gift-strategy
chmod +x ./start-server.sh
./start-server.sh
```

启动后打开下面地址验收：

`https://gift-strategy.run.zhenguanyu.com/?period=2026-618`

## 三、后续更新线上

本地或 GitHub 改完并 push 后，登录 Run 机器执行：

```bash
cd /home/shared/workspace/gift-strategy
git pull
chmod +x ./start-server.sh
./start-server.sh
```

然后刷新线上页面验收。

## 四、注意事项

- 不要上传 `env.md`、`id_ed25519`、`server.log` 这类凭证或日志文件。
- 图片路径是相对路径，`data/periods.json` 里写的图片必须真实存在于 `assets/`。
- 浏览器三击编辑后，要用 `Ctrl+Shift+E`（Mac：`Command+Shift+E`）导出 JSON，再提交到 GitHub。
- 如果线上看不到最新内容，先确认 GitHub 已 push，再确认服务器执行过 `git pull` 和 `./start-server.sh`。
